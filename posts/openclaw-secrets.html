How I Stopped Sprinkling Tokens Everywhere in OpenClaw

SecretRef where it fits. Runtime materialization where it doesn’t.

If you self-host OpenClaw, token sprawl creeps in fast: one value in config, one in systemd, one in a helper script, one in a forgotten .env. This post is the practical fix.

> the_short_version

  • Use SecretRef for credential paths that natively support it.
  • Use runtime materialization for string/file consumers.
  • Keep a canonical secret store (e.g. Bitwarden Secrets Manager), not scattered plaintext.
  • Validate every change with doctor + secrets audit.
